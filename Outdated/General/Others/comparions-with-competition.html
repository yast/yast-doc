YaST2: Comparison with other admin tools


In the linux community there are currently two larger projects tackling the problem of system configuration and administration.

  • linuxconf
  • coas

Both have prominent supporters, namely RedHat (linuxconf) and Caldera (coas), but neither has made a larger impact in the area of system administration.

Pros

linuxconf

  • rich set of features
  • modules (which use a common API)
  • multi language
  • basic network support (admin groups)
  • curses, X, www (via java) interface

coas

  • schema definition to describe data formats
  • curses, X, www (via java) interface

YaST2

  • workflow (desribing the what and the how)
  • component system (building complex structures)
  • batch processing (automated configuration)
  • client/server concept

Cons

linuxconf

  • 40000 lines of C++ code
  • focus on config file editing and restarting daemons

coas

  • new and rather unsupported
  • focus on data representation
  • sparse documentation
  • only C++ and python
  • very limited in version 1.0

YaST2

  • yet another attempt on administration
