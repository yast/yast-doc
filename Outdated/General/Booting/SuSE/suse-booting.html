YaST2: Analysis: Booting: SuSE Linux

Scope

This document describes the initial boot of SuSE Linux from a boot disk or CD-ROM. It also explains, why the setup was choosen.

Disk Format

The boot disk is MS-DOS compatible, so users can 'see' the disk under DOS and Windows. Having a ext2fs (linux standard) or minix (most efficient) filesystem will increase load on I-Support with people complaining about 'empty' disks.

This filesystem also allows exchanging the kernel by copying a different kernel image from CD-ROM to disk on a Windows system.

Boot loader

Syslinux was choosen instead of lilo for it's support of the DOS filesystem. Lilo needs to know the physical location of the kernel on the disk, hence the need to call 'lilo' after installing a new kernel.

Syslinux also supports a 'boot up' message to have an initial screen for the user.

Ram Disk Format

The initial ramdisk (initrd.gz) and the modules (scsi-mod.gz and net-mod.gz) have minix filesystem. This is the most efficient filesystem and can be fine-tuned to lowest overhead. The minix fs module is also the smallest.
