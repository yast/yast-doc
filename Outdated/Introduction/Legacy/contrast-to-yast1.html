YaST2: RSpec: YaST1

This chapter briefly positions YaST2 in contrast to YaST1.
YaST1 is a widely accepted and respected solution to the installation and administration requirements of a linux distribution. However, it tries to be everything to everyone combining all tasks in a single program.

This "one-for-all" approach is currently the biggest obstacle in keeping YaST1 in sync with the requirements of a modern linux distribution. The code has grown over the past years to a state where it's almost unmanageable. Currently only two people have knowledge about the code and they're the loudest in crying "don't touch it and don't use it".

The YaST1 interface is respected and most SuSE customers won't abandon it lightly. For that very reason, YaST2 must not break with the interface concept from YaST1.

The most often heard demand for YaST1 is a graphical (point-and-click) interface. At least, using the mouse on the current text-based interface should be possible. Due to instabilities which may be introduced by such an extension, the YaST1 maintainers refuse to add this functionality.

Another important point is the shift of the target market for SuSE Linux. This moves away from the current programmer in two directions: Windows users and corporate users.
The first needs an easy installation, the second needs an easy (and network-capable) administration. Both issues are not addressed by YaST1.

For YaST2, we must focus on

    the customer market (most important)
      graphical interface
      automatic installation
      automatic recognition of changed/added hardware components
      extensive, context-sensitive help
      internationalisation

    the corporate market
      remote administration
      group and cluster administration
      logging, recording, and playback of actions
      security (authorization, encryption)

    a documented and maintainable tool
      highly modularized software
      clear interfaces
      easily upgradable

    quality assurance
      automatic testing
      regression testing
      multi-platform testing

    separation of program and interface
      interface fits to environment (gnome, kde, ...)
      interface look and feel easily changable without interference with program
      remote interface
      web based interface
