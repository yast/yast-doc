What is a YaST II module?

There are different types of YaST II modules:

  • Program modules
  • Process modules
  • Plugin modules
  • Object file modules
  • Script modules

Currently only the first type (program module) and the last type (Script modules) are being used. The different module types differ in how they are called and how the communicate with other components.

Program modules

A YaST II program module is an executable UNIX binary with the following properties:

  • It can be given an arbitrary number of arguments. Each argument has YCP ASCII syntax.
  • When called it issues an ASCII representation of a YCP value to stdout. Now there are two possibilities:
    • If that value has the form result(value), i.e. a term with the symbol result and exactly one argument, the module terminates immediately afterwards and thus closes its stdout and stdin. value is considered the be the result, return value or exit code of the module (whichever name for it you like best).
    • If that value has any other form, the module reads on stdin and wait for an YCP value in ASCII representation. The module assumes that the value it has issued is now interpreted by the user interface and waits for an answer. When it has read one YCP value on stdin it considers this to be the answer of the user interface. Now the whole loop starts over again and the module issues the next value on stdout.
  • The program does not terminate without first having issued result(value)
