Thrown into something very complex

The design of software is a very complex work. Be aware that style guides are not cookery books with detailed lists of ingredients you can put together without thinking. Style guides are always "only" guidelines. You have to use them in a intelligent manner. Sometimes you may be forced to disobey them. But then do it clever and provide a good explanation for your decission.

Goals for this style guide

  • Build up a consistent YaST2 - user should realize: YaST2 was made by one man/woman

  • Integrate YaST2 into the corporate design of SuSE

  • Make the living with SuSE Linux easy for the user

  • Make the development of YaST2 easy for the developer

The dark force in the background - international norms

Every software - even YaST2 - should persist a check under following aspects (this is a short overview about the demands of ISO 9241 Part 10: Dialogue principles):

A 'copy' of the German DIN EN ISO is available in the appendix (***work in progress, English in press***.

Suitability for the task (German: Aufgabenangemessenheit):

  • Software is a tool to work with. On work is less time for "gaming" and trouble shooting. Users must be able to accomplish their goal with minimal effort and maximum results and the system must not treat the user in a hostile fashion or treat the user as if they do not matter (T. Burgess).

  • ISO quintessence:

    • Die Software ist auf die Arbeitsanforderungen zugeschnitten und ermöglicht somit effizientes Arbeiten:

      • Die vorgegebenen Arbeitsschritte sind nötig und sinnvoll

      • Die Arbeitsaufgaben können ohne Umwege abgearbeitet werden

      • Arbeitsaufgaben können vollständigerledigt werden.

    • Die Interaktion mit der Dialogschnittstelle ist auf die Aufgabenstellung abgestimmt:

      • Unterschiedliche Arbeitsaufgaben können mit angepaßten teraktionsmodi abgearbeitet werden

      • Die verwendeten Begriffe und Bezeichnungen entsprechen der Arbeitstätigkeit

      • Wichtige Befehle sind schnell zu finden

    • Die Informationen am Bildschirm sind so angeordnet, daß die Aufgabenbearbeitung unterstützt wird

      • Benötigte Informationen werden angezeigt und passen zur Aufgabenstellung.

      • Die von der Software erzeugten Datenausgaben auf dem Bildschirm, Drucker, Plotter etc. lassen sich an die unterschiedlichen Aufgabenstellungen anpassen.

      • Für sich wiederholende Arbeitsschritte gibt es eine Wiederhol-Funktion.

  • Our work for YaST2:

    • YaST2 should be optimized to intstall and configure SuSE Linux



Self-descriptiveness (KDE: understandable, German: Selbstbeschreibungsfähigkeit):

  • When the user uses the application for the first time, he should be able to see quickly what it does and how to use it (KDE style guide).

  • ISO quintessence:

    • Clear functionality of the dialogs

      • Information about 'what can I do now' and

      • 'What will happen if I do this...'

    • Clear feedback to the user

      • The feedback for the user reflects the actual situation (content?) of the work progress.

      • Es ist klar und eindeutig, wo und in welchem Kontext ich mich befinde und wo die aktuelle Interaktionsstelle zu finden ist (z.B. durch Markierung,Farbe, Cursorblinken, Mauscursor etc.)

      • Rückmeldungen, Sicherheitsabfragen, Warnungen oder Fehlermeldungen sind klar unterscheidbar

    • Clear use of text information

      • Der Inhalt der Hilfe ist unmittelbar verständlich

      • Bei Bedarf kann man zur Situation weitere Erläuterungen anfordern. Diese Funktion ist so oft wie es geht (d.h. Immer) verfügbar und schnell und leicht zu benutzen

      • Es werden auf Anforderung auch Beispiele vorgestellt und Übersichtsinformationen über die aktuellen Bedien- und Nutzungsmöchkeiten aufgezeigt.

      • Aviod abbreviations in the dialogs

    • Our work for YaST2

      • Use well known standards based on ergonomic research

      • Display only the information important to the user and mask unimportant information

      • Make a additional help button available



Controllability (KDE: navigable and feedback rich, German: Steuerbarkeit):

  • The user should always be able to tell where he/she is. Don't restrict navigation too much but avoid such things like "lost in Hyperspace". There must be constraints on the users actions (T. Burgess).

  • ISO quintessence

    • User has control

      • User can make self determined pauses

      • No force to keep a strictly determined workflow

      • User can stop working on a process and start at the same point later

    • Comfortable navigation is possible

      • Even jumps between different areas are easy an quick

      • To all time, there is the opportunity to 'Abort' the operation

      • The software supports different interaction modes (keyboard, mouse, menu, key short cuts)

  • Our work for YaST2

    • The application should always give immediate feedback to the user regarding which actions(s) are being taken. And make it possible to use different interaction modes like mouse and keyboard.

    • Think about a history function.

    • Aviod information overload.



Conformity with user expectations (KDE: conformable to expectations, German: Erwartungskonformität):

  • The application should be consistent throughout!

  • ISO quintessence

    • The dialogs and all dialog elements have a consistent design and consistent use

      • Same functions are located at same places

      • Software feedback has it's own place

      • The function of the dialogs and dialog elements is predictable

      • Consistent naming of objects and labeling of function elements

    • Time prognoses are available

  • Our work for YaST2

    • A table list is a table list and looks always similar. A "Yes-No" message box looks always similar too.

    • The "Next" button is on the right side and the "Back" button on the left, in western countries. And japanese or hebrew people may not agree to this.

    • The print menue is always the same, even if it was activated from different dialogues.



Error tolerance (KDE: tolerant of mistakes, German: Fehlertoleranz):

  • Users are human - they make mistakes. The system can not crash or produce any unexpected results at any point in the process (T. Burgess).

  • ISO quintessence

    • Fehler haben keine schwerwiegenden Folgen

      • Fehler können leicht rückgängig gemacht werden (undo)

      • Befehle, die Daten unwiderruflich löschen, sind mit einer Sicherheitsabfrage gekoppelt.

      • Die Originaldaten können z.B. durch ein Backup oder ein undo wiederhergestellt werden

      • Man kann Möglichkeiten ausprobieren ohne Angst zu haben

    • Fehler werden von vorneherein vermieden

      • Eingaben des Benutzers, werden auf ihre Stimmigkeit/Richtigkeit hin überprüft bevor die Verarbeitung stattfindet

      • Systemfehler (z.B. "Absturz") treten bei meiner Arbeit mit der Software nicht auf.

      • Versehentliches Auslösen kritischer Aktionen ist durch die Dialoggestalltung erschwert.

    • Es gibt konkrete Informationen über die Fehlerbewältigung

      • Diese Informationen sind verständlich

      • Die Software warnt vor Fehlern, bevor eine Aktion ausgelöst wird

    • Our work for YaST2

      • The application should allow them to do multiple Undos.

      • It is impossible to unformat a new partitioned and formated hard disk, so provide a clear help and built in security checks.



Suitability for individualisation (German: Individualisierbarkeit):

  • People think and act different so be prepared for varying habits to use a interface.

  • ISO quintessence

    • Allgemeine Individualisierung

      • User can adjust the general design of the screen layout

      • Die Software kann einfach an den individuellen Kenntnisstand angepaßt werden (wegnahme bzw. Hinzufügen bestimmter Informationen und Steuermöglichkeiten)

      • Die Eingabeeigenschaften der Geräte sind einstellbar

      • Die Verwendung von Macros ist möglich

      • Die Hilfe kann individuell eingestellt werden

      • Die Reaktionszeiten des Systems können individuell geregelt werden

    • Arbeitsspezifische Individualisierung

      • Arbeitsrelevante Eingabemasken und Bildschirmdarbietungen sind anpassbar, der Arbeitsfluß ist individualisierbar. Darunter fallen z.B. auch Menü-Einstellungen.

      • Datensatzfelder, Kommandos, Objekte, Aktionen können z.B. Individuell benannt werden.

  • Our work for YaST2

    • Prepare more than one possibilities to control the software. Some like it quick and simple, others like it sophisticated.




Suitability for learning (German: Lernförderlichkeit):

  • Perhaps we can manage this difficult task - to teach the user something about SuSE Linux during the install process.

  • ISO quintessence

    • Quick and easy learning should be possible

      • Short training time

      • Good recognition after a long time of not using the software

      • No information overflow during use of Software

    • User can explore the software

  • Our work for YaST2

    • Make a good help text.

    • Echo the command actually run



To outline this point of view

Not all users are hackers! Normaly the programmer has to create a software with great

  • usability

  • utitily

  • and fun factor.

Be aware of the different needs by different users

Hackers want

Managers want

Users want

New advanced technology system as a challenge and oportunity to tinker [?] with

A bussines solution system with stability, return on investment and controlled release cycles

Usability to use, utility for work and fun



Outcome of low usability

  • More time an greater effort for the user to learn use of the system

  • More time for the user to finish a task

  • Less quality (e.g. more errors) in the outcome of the task

  • More emotional and cognitive stress for the user and linked to this topic more physical stress

  • More costs for the employer

  • Less image for the manufacturer of the software

  • Less sold software boxes
