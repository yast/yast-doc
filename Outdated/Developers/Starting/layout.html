YaST2 Developers pages: Directory layout

The general directory layout is

package/toplevel dir
package/docdocumentation (manually edited)
package/doc/autodocsdocumentation (ydoc generated)
package/srcsources
package/src/includeinclude files
package/packagepackage files for autobuild
package/testsuitetestsuite

A module must have a MAINTAINER file which states the person responsible for the module.
The (SuSE package compatible) module name must be defined in RPMNAME, the version must be defined in VERSION in a X.Y.Z format.

Configuration and compilation must be done with autoconf and automake, just look at other modules on how to use these tools.
