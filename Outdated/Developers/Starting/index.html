YaST2 Developers pages: Getting started

Welcome as a YaST2 developer.


To get started with hacking on YaST2, you should subscribe yourself to the yast2-hacker mailing list. Send a mail to SuSEdomo with subscribe yast2-hacker in the body of your mail. You might also use the Web interface.

The next step is to get access to the YaST2 CVS repository. It's not a requirement, but it's easier to exchange code with other developers and good for backups.
Read and understand the YaST2 CVS introduction and the YaST2 CVS guidelines. The latter is especially important in a shared development enviroment.

How to get started

  • Think before you code

    Before you start hacking, start thinking. Read and understand the usability rules and design your interface according to the YaST2 style guide.

    If your module needs several steps to accomplish its task, it is a workflow. There is a separate document which gives some guidance for the design of workflows.

  • So you want to write a module ...

    The y2c_skeleton (below yast2/modules) is a good template. Ask Michael Hager on how to use it.
    For information on how to code the interface, look at source/libyui/examples from the CVS.

  • Documenting it all

    This doc howto is a bit outdated but still valid.

  • Understanding the directory layout

    Every module (source) directory in YaST2 should adhere to a common layout.

  • Compiling the source

    If you want to compile the complete C++ source code (usually not needed), read here.

  • Preparing a package for autobuild

    Preparing a SuSE compatible package is quite easy, as make package does everything you need.

Testing and Debugging

There are some tools to test and debug your code. If you have libycp installed, parseycp will provide a quick syntax check.
Pure ycp code (just to get used to ycp in general) can be run with runycp, complete modules are better checked with
/usr/lib/YaST2/bin/y2base module.ycp qt
Setting the Y2DEBUG environment variable makes YaST2 really verbose. The package libycp provides more information at /usr/share/doc/packages/yast2/libycp/logging.html

An extremely good approach is to include automatic testing with the dejagnu package. This document explains the necessary setup. Also look at yast2/source/libycp/testsuite (from the cvs repository) for a real-world example.
