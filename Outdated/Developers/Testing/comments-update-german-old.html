Anmerkungen zum Update und zur Entwicklung der Checkliste



Vorbemerkungen

Die Verwendung des Begriffs 'Update' bezieht sich auf das Update des gesamten Systems.

Ausgangspunkt für ein Update ist ein stabiles, funktionierendes SuSE-System (d.h. das Update führt keine Reparatur des Systems durch). Es existieren keine Plattenplatzrestriktionen und es liegt eine gültige rpm-Datenbank vor.

Wichtig: Die Linux-Version die einem Update unterzogen wird, ist eine CD-Installation aus einem Verkaufspaket oder wurde von /dist installiert. Es ist eine genaue Dokumentation nötig, die die Version beschreibt.

Außerdem: Das Testsystem sollte im Idealfall kein frisch installiertes Standard SuSE LINUX sein, sondern individuelle Einstellungen enthalten. Ziel ist es, zu überprüfen, ob individuelle Einstellungen korrekt übernommen werden.



Was macht das Update?

Es werden während des Updates alle installierten Pakete überprüft und anhand des Datums-/Versionsstempels für das Update ausgewählt.

  1. Check der installierten rpms und Festlegung der Reihenfolge der Abarbeitung der einzelnen Pakete (ggf. manuelle Auswahl bei Unklarheiten).

  2. Backup von Dateien, deren Datumsstempel nicht denen in den rpms entspricht.

  3. Abarbeitung der einzelner Updates der rpms mit 'rpm -U...'
    -> Die Paketbauer haben aufgrund der Vorgehensweise eine besondere Verantwortung für das Update.

  4. SuSEconfig und lilo werden kontrolliert



Hinweise zur Dokumentation:

Die nötigen Dokumente findet man im Moment

  • im YaST2-CVS-Tree unter /doc/public/Developers/Testing/*

  • und im Exportverzeichnis von sigi unter http://w3.suse.de/~sigi/Update_test/*

Es gibt zwei Dateien:

  • comments-update.html = (diese) Konzeptionsdatei

  • checklist-update.html = Checkliste für das Update

Hinzu kommen die jeweiligen *.ps-Dateien für einen schönen Ausdruck. (Die Dokumente haben im Moment den html-Code von StarOffice = viel Müll) und es gibt Probleme beim Ausdruck mit Netscape oder Konquerer.



Konkretes Vorgehen beim Update:

Bitte haltet euch an die Checkliste. Diese ist V1.0 und muß natürlich noch überarbeitet werden. Kommentare hierzu sind sehr willkommen. Es ist wichtig, daß die Tests praktikabel sind (aufwendig sind sie sowieso). Die ausgefüllten Formulare könnt ihr bitte bei Sigi oder Jörg vorbeibringen (4-N-5).

Falls Fehler auftreten könnt ihr sie gleich in Bugzilla eintragen. Hierzu haben wir eine neue Gruppe 'SuSE Linux/Update Problems' eingerichtet.

Schubi hat - damit jeder dieselben Updatefiles nimmt - auf einem Rechner eine extra 7.0-Distribution gelagert. Diese muß im Moment für das Update benutzt werden.

  • Bequemes Booten mit der 1. CD der 7.0; YaST2 startet -> laufen lassen und dann bei der ersten Gelegenheit mit dem Button 'Installation abbrechen' in das Linuxrc springen (ist beim Booten von fd0 natürlich nicht der Fall).

  • Linuxrc startet

    • 1. Kernel module (auto-)laden

    • 2. Start installation/system

    • 3. Start installation/update

    • 4. Network (NFS) -> Automatic configuration via bootp 'Yes'

    • 5. IP address of name server '10.10.0.1'

    • 6. NFS-Server: 10.10.1.140

    • 7. directory: /test-update/full-i386

    • 8. Choose version of YaST: YaST2 - graphical

    • 9. YaST2 müßte jetzt vom gemounteten System starten und funktionieren

    • 10. Im Installations-Schritt 2 (Caption: 'SuSE Linux installieren') ->'Update YaST2 (only beta-version!!!)' auswählen

Falls die root-Partition nicht klar ist (z.B. bei vielen versch. Systemen auf einem Rechner) gibt es einen Frage-Dialog.

Nochmal: Herauszustellen wäre noch, daß es wichtig ist, daß das 'alte' System kein frisch installiertes Standardsystem sein sollte, sondern ein System mit individuellen Einstellungen und Veränderungen.



Anmerkung zur Datensammlung:

Updates werden ab SuSE-Linux 6.2 getestet. Besonderes Augenmerk liegt auf 6.3->7.0 und 6.4->7.0 (bzw. 7.1).

Bitte schiebt die gespeicherten Protokoll- und Logfiles in ein extra Verzeichnis. Jörg Dippel (jd) hat für die Speicherung hierfür einen extra Platz eingerichtet:

  • /suse/jd/updatetest/<datum-Rechnername> (z.B. /suse/jd/updatetest/20001023-schoenwares)
