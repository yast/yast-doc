Abstract

This document is a checklist for tests to check a new installation using YaST2.


General information

General information is needed to identify problems. First whether there is a problem but also second if a test has succeded but there is still a problem, to identify the differences, why the problem doesn`t occurs nearby the tested system.

Short hardware description

Several problems are related to a specific hardware. Hardware information is needed to identify problems. If a problem occurs you should mention that specific hardare.

Please have a look to the output of the hardware logfile. Check it whether the identified hardware correlates to your system.

To evaluate the test coverage of configured hardware four hardware descriptions are named explicitely: these are graphics, sound, printer, connection via modem/ISDN. The description is only necessary, if the hardware is available. The information can be retrieved from hwinfo output.

Hardware description

Save output of hwinfo

Hwinfo is a small program to identify hardware. To use it, run it with a command line similar to this:

hwinfo-0.90-ix86 log=hw_log

You can get more information about that program using ftp://ftp.suse.com/pub/people/snwint/hwinfo/
