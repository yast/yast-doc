YaST2: Resources: Framebuffer: Changing the Logo

This document describes how to change the boot-up graphic displayed when booting with the framebuffer device.

  1. Overview

    Before using the framebuffer device a new kernel has to be compiled -- the console drivers have to be configurated.

    Then on booting the system a small tux logo of the size 80x80 is displayed.

    This document describes what has to be done to exchange this image.

  2. Purpose and Scope

    This document doesn't describe what changes are necessary within the code for an image with 8 bit color depth using packed format. This is already done within that document.

    This document describes how an images must be transformed to be displayed.

  3. The Graphic Format

    An image is described by a color map. This color map holds 214 colors of 24 bit color depth, 8 bit for each color channel of the RGB color space.

    A packed image of depth 8 bits using color registers (color map) is a image, within every pixel is described by exactly one index that points to the color map.

    Clearly spoken, each value within the picture definition represents one color, identical index values are representing identical colors.

  4. The Transformation

    The easiest way is to transform the image to an XPM format. For this procedure xv can be used.

    The XPM format contains information about the picture size and also about the number of used colors.

    Only if the restriction holds that less or equal 214 colors are used, the image definition can replace the tux logo.
    Otherwise the number of the colors have to be reduced, maybe by using a color metrik.

    The XPM format contains a list of colors and their representation within the 24 bit RGB color space. This information can taken directly to fill the color tables: linux_logo_red[], linux_logo_green[] and linux_logo_blue[]. The sequence of the definitions is arbitrary.

    The XPM format contains also a mapping table of printable ASCII characters which are representing an index -- such a string is representing a 24 bit color.

    The image itself is defined line per line. This means the sequence is defined -- but it is the same sequence like in the linux_logo[].

    To get the replacement logo the printable ASCII character string has to be replaced by the index representing the color within the color maps.

    But pay attention, the colormap index starts at 32, the first 32 colors are reserved. This means a value of 32 within the image definition means the first entry within the color maps!

  6. Generation Infos 

    # call make image=imagefile
#           imagefile is a filename without extention !

special:
	gcc -DXPM_IMAGE=\"$(image)\" -DXPM_NAME=$(image) image.c

    To transform the image suse_150.xpm, 'make image=suse_150' is called.
    The resulting program a.out has to be executed.

    Then the result, which is displayed on stdout, has to be integrated into linux_logo.h.
